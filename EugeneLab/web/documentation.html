Comments

miniEugene supports line comments.

Example:

// This is a line comment

Disjunction of Constraints

miniEugene supports the conjunctive normal form, i.e., a conjunction of disjunction. Line feeds indicate a conjunction. In one line, it is possible to specify a disjunction of constraints.

Examples:
1. The required design must contain a, d, and b or c.

CONTAINS a
CONTAINS b OR CONTAINS c
CONTAINS d
2. All components in the design must be either forward or reverse oriented.
ALL_FORWARD OR ALL_REVERSE

Templating Constraints

The provided constraints in miniEugene are not expressive enough to specify various types of synthetic biology designs. For example, consider the design 

p, r, c, t, p, r, c, t
Therefore, we augmented miniEugene with Templating Constraints.

The EBNF of Templating Constraints:

templating            = templating_constraint list_of_ids
templating_constraint = (name)? tconstraint_operator list_of_parameters
name                  = [a-zA-Z]+
tconstraint_operator  = ["template"|"TEMPLATE"|"sequence"|"SEQUENCE"]
list_of_parameters    = ( "[" selection "]" | id ) ("," list_of_parameters)?
selection             = id ( "|" selection )?
Templating Constraints are divided into:
  • A Template Constraint defines a universally valid pattern. Template constraints require that the Length of the design is a multiple of the length of the template constraint. For example, a 
    template p, r, c, t
    constraint and a length of 12 (3 * 4) provides the following design:
    p, r, c, t, p, r, c, t, p, r, c, t
    If the provided length of the design is not a multiple of the length of the template constraint, then an exception is thrown.
  • A Sequence Constraint defines an ordered set of components that must occur in a particular order.
    For example, a 
    sequence p, r, c, t
    constraint and a provided length of 6 results in the following set of designs ('_' denotes place holders):
    p, r, c, t, _, _
_, p, r, c, t, _
_, _, p, r, c, t

It is, of course, possible to utilize miniEugene's templating constraints in conjunction (and disjunction) with all other provided constraints. See the Examples section for more detailed examples.

Note:

  • For the time being, Templating Constraints do not allow features to constrain the orientation of the components constraint by a templating constraint. Please use the Orientation constraints.

Constraints

We categorize the miniEugene constraints into the following types of constraints: counting, positioning, pairing, orientation, and interaction.

Pairing Constraints constrain the pair-wise occurrence of genetic elements.

[0] EQUALS [1]

Interaction Constraints

Interaction Constraints constrain (regulatory) interactions and relations among components in a genetic design.

Constraint Description Examples

α REPRESSES β

If α and β appear in the design, then the α and β have a REPRESSES relationship.

tetR REPRESSES pTet

α INDUCES β

If β appears in the design, then the α INDUCES the β component.

aTc INDUCES pBad

α DRIVES β

If α and β appear in the design, then α and β have the same orientation, α occurs upstream of β, and between α and β does NOT appear any terminator. We say, α DRIVES the expression of the β.

Promoter DRIVES Repressor

pTet DRIVES GFP
