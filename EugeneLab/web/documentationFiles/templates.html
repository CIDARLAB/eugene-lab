Templating Constraints

The provided constraints in miniEugene are not expressive enough to specify various types of synthetic biology designs. For example, consider the design 

p, r, c, t, p, r, c, t
Therefore, we augmented miniEugene with Templating Constraints.
A selection provides a list of possible ids. The selection operator ("[ ]") selects a single id from the provided list.

Templating Constraints are divided into:

  • A Template Constraint defines a universally valid pattern. Template constraints require that the Length of the design is a multiple of the length of the template constraint. If the provided length of the design is not a multiple of the length of the template constraint, then an exception is thrown.
    Examples:
    • With a given length of 12 (3 * 4), the template constraint 
      template p, r, c, t
      produces the following design:
      p, r, c, t, p, r, c, t, p, r, c, t
    • With a given length of 1, the template constraint 
      template [p|c]
      produces the following designs ('-' denotes reverse orientation):
      p
-p
c
-c

  • A Sequence Constraint defines an ordered set of components that must occur in a particular order.
    Examples: 
    • sequence p, r, c, t
      constraint and a provided length of 6 results in the following set of designs ('_' denotes place holders):
      p, r, c, t, _, _
_, p, r, c, t, _
_, _, p, r, c, t
    • The selection operator can also be utilized within sequence constraints. With a given length of 1, the 
      sequence [p|r|c|t]
      sequence constraint enumerates the following designs ('-' denotes reverse orientation):
      p
-p
c
-c

It is, of course, possible to utilize miniEugene's templating constraints in conjunction (and disjunction) with all other provided constraints. See the Examples section for more detailed examples.

Note!

  • Currently, Templating Constraints are not efficiently implemented. As soon as we've solved this problem, we'll announce it properly on this website.
  • For the time being, Templating Constraints do not allow features to constrain the orientation of the components constraint by a templating constraint. Please specify additional Orientation constraints.
